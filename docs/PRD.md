# 微型圖書館 (Mini Library) 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
在小型社群、辦公室或家庭中，書籍的流通往往缺乏系統化的管理，導致書籍遺失、借閱狀況不明確或歸還效率低下。為了提高書籍的利用率並簡化管理流程，「微型圖書館」系統旨在提供一個輕量、直觀的平台，讓借閱者能輕鬆找書、管理者能高效維護館藏。

### 目標用戶
- **借閱者 (Borrower)**：希望快速搜尋書籍、了解借閱狀態並追蹤自己的閱讀歷史。
- **管理員 (Administrator)**：負責維護圖書清單、處理借閱/歸還流程以及追蹤逾期情況。

### 核心價值主張
- **透明化**：即時掌握每本書的在庫狀態。
- **效率化**：簡化借還流程，透過提醒機制降低逾期率。
- **數位化**：取代傳統紙本紀錄，讓借閱歷史有跡可循。

## 2. 功能需求

### 2.1 主要功能與使用者故事

| 功能模組 | 使用者故事 |
| :--- | :--- |
| **圖書目錄管理** | 作為 **管理員**，我希望 **能夠新增、編輯與刪除圖書資訊**，以便 **維持館藏資料的正確性**。 |
| **借閱與歸還流程** | 作為 **借閱者/管理員**，我希望 **能透過系統紀錄借書與還書動作**，以便 **即時更新書籍的在庫狀態**。 |
| **狀態追蹤與提醒** | 作為 **管理員**，我希望 **能篩選出所有逾期未還的書籍**，以便 **主動提醒借閱者歸還**。 |
| **書籍搜尋與過濾** | 作為 **借閱者**，我希望 **能根據書名、作者或分類搜尋書籍**，以便 **快速找到想讀的書**。 |
| **個人紀錄查詢** | 作為 **借閱者**，我希望 **能查看我的借閱歷史與當前借書清單**，以便 **了解自己的閱讀進度與歸還期限**。 |

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask。
- **前端模板**：使用 Jinja2 搭配原生 CSS/JS。
- **資料庫**：使用 SQLite 進行本地資料儲存。

### 效能與安全考量
- **響應式設計**：介面需適配手機與桌面端，方便隨時操作。
- **簡易權限管理**：區分管理員權限（管理圖書、查看所有紀錄）與一般用戶權限（搜尋、查看個人紀錄）。
- **資料完整性**：確保借閱與歸還邏輯嚴謹，避免一書多借或狀態錯誤。

## 4. MVP 範圍

### Must Have (第一階段必備)
- 圖書基本資訊維護 (CRUD)。
- 簡易的借閱與歸還功能。
- 當前書籍狀態顯示 (在庫/已借出)。
- 基礎的搜尋功能（依書名搜尋）。

### Should Have (第二階段應有)
- 逾期書籍標記與清單匯出。
- 使用者帳號登入系統。
- 個人借閱紀錄列表。

### Nice to Have (未來優化)
- 郵件或系統推播逾期提醒。
- 書籍評論與評分功能。
- 書籍封面圖片上傳與顯示。
