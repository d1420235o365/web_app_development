# PRD — 食譜收藏系統

> 版本：v1.1　　最後更新：2026-05-07　　狀態：草稿

---

## 1. 專案概述

### 1.1 背景與動機

現代人常因為不知道吃什麼而煩惱，或是手邊有剩餘食材卻不知道如何應用。此外，網路上食譜散落各處，難以有系統地集中管理與搜尋。本系統旨在打造一個功能完整的食譜收藏網站，讓使用者能輕鬆儲存、搜尋並根據自身需求找到最適合的食譜，同時針對健身者、孕婦等特殊族群提供專屬功能。

### 1.2 目標用戶

| 角色 | 描述 |
|------|------|
| **一般使用者** | 希望儲存、瀏覽與搜尋食譜的家庭主婦、料理愛好者 |
| **健身者** | 需要根據熱量、蛋白質等營養素評估食譜是否符合飲食計畫 |
| **孕婦／產後媽媽** | 需要依特定營養需求（葉酸、鐵質等）篩選安全且適合的食譜 |
| **管理員** | 負責審核、管理平台上的食譜內容，確保資訊正確性 |

### 1.3 核心價值主張

- **集中管理**：一站式儲存與瀏覽個人食譜收藏
- **智慧搜尋**：依食材、分類、難度快速找到適合食譜
- **族群專屬**：針對健身者、孕婦等特殊需求提供個人化推薦
- **減少剩食**：從手邊食材出發，智慧推薦可做料理

## 2. 功能需求

### 功能清單與使用者故事

#### F01 — 儲存食譜

作為**一般使用者**，我希望能新增食譜（包含名稱、食材、步驟、圖片、難度等資訊），以便將自己喜歡的料理完整保存在系統中。

- 填寫食譜名稱、描述、食材列表、烹飪步驟
- 上傳食譜封面圖片
- 設定難度（簡單 / 普通 / 困難）、烹調時間、份量
- 選擇分類標籤（早餐、主食、甜點、健身、孕婦等）
- 設定公開或私有權限

---

#### F02 — 瀏覽與搜尋食譜

作為**一般使用者**，我希望能依關鍵字、分類、難度快速搜尋食譜，以便在眾多食譜中找到我想做的料理。

- 關鍵字搜尋（食譜名稱、描述）
- 依分類標籤篩選
- 依難度等級篩選
- 依烹調時間排序
- 分頁顯示搜尋結果

---

#### F03 — 從食材組合尋找食譜

作為**一般使用者**，我希望輸入我手邊現有的食材，系統能推薦可以製作的食譜，以便充分利用冰箱食材，減少浪費。

- 輸入多種食材（以標籤方式輸入）
- 系統回傳「完全符合」與「部分符合」的食譜
- 顯示每道食譜缺少的食材清單
- 依符合程度排序顯示結果

---

#### F04 — 健身者專用飲食評估食譜

作為**健身者**，我希望能查看食譜的詳細營養素資訊（熱量、蛋白質、脂肪、碳水化合物），並根據我的健身目標（增肌 / 減脂 / 維持）篩選適合的食譜，以便讓飲食計畫更科學有效。

- 食譜詳情頁顯示完整營養標示（每份）
- 提供「增肌推薦」「減脂推薦」等專屬篩選標籤
- 根據設定目標，系統標記食譜是否適合
- 顯示高蛋白、低卡等健身友善食譜排行

---

#### F05 — 孕婦及產後媽媽專用食譜

作為**孕婦或產後媽媽**，我希望能篩選含有特定重要營養素（如葉酸、鐵質、DHA、鈣質）的食譜，並排除懷孕期間應避免的食材，以便安心選擇適合自己的料理。

- 專屬「孕婦友善」「哺乳期適用」分類標籤
- 標示每道食譜的重點營養素
- 自動警示含有孕期禁忌食材的食譜（如生魚片、酒精等）
- 提供孕期各階段（初期 / 中期 / 晚期 / 產後）的推薦食譜

---

#### F06 — 使用者註冊與登入

作為**一般使用者**，我希望可以註冊專屬帳號並安全登入，以便擁有獨立的食譜收藏空間，並跨裝置存取個人資料。

- 使用者註冊、登入與登出
- 密碼使用 bcrypt 雜湊加密儲存
- Session 管理，保持登入狀態

---

#### F07 — 管理員後台管理

作為**管理員**，我希望能在後台審核、編輯或下架不適當的食譜，以便確保平台內容的品質與安全性。

- 管理員專屬登入頁面
- 瀏覽所有食譜列表（含審核狀態）
- 核准、退回或強制下架食譜
- 編輯任意食譜內容
- 管理使用者帳號

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML5 + CSS3 + 原生 JavaScript |
| 部署環境 | 本地開發（可延伸至雲端） |

### 3.2 效能考量

- 食譜列表頁每頁顯示 12 筆，避免一次載入過多資料
- 食材搜尋應在 1 秒內回傳結果
- 圖片上傳需限制大小（最大 5MB），並壓縮儲存
- 資料庫索引設計良好，確保食材組合搜尋效能

### 3.3 安全考量

- 管理員介面需驗證 Session，防止未授權存取
- 表單輸入須進行 XSS 防護（使用 Jinja2 自動 escape）
- SQL 查詢使用參數化查詢，防止 SQL Injection
- 圖片上傳需驗證副檔名，避免惡意檔案上傳
- 密碼使用 bcrypt 安全雜湊加密
